Идея

Grammar Steg прячет произвольные данные внутри обычного русского текста. Секретное сообщение превращается в последовательность реальных предложений из большого корпуса. Для постороннего читателя это выглядит как связный текст; при знании алгоритма из него можно восстановить исходные байты.

Кодирование

  1. Входные данные. Можно передать UTF-8 текст, бинарный файл или уже готовый поток бит. Данные переводятся в последовательность нулей и единиц (по 8 бит на байт).
  2. Опциональное шифрование. Можно зашифровать симметричным паролем (gpg --symmetric) или публичным ключом получателя. В обоих случаях в предложения встраивается компактный бинарный OpenPGP-шифротекст (AES-128, ZIP), без ASCII armor — чтобы не раздувать cover-текст.
  3. Перемешивание бит. … Одинаковый вход и версия алгоритма всегда дают одинаковый результат. Длина потока бит не меняется.
  4. Выбор предложений. Обработанные биты читаются блоками фиксированной длины (… бит на предложение). Каждый блок — это номер предложения в корпусе. В конец добавляется служебный бит-маркер, чтобы при декодировании понять, где заканчиваются данные.
  5. Оформление текста. Выбранные предложения склеиваются через пробел и группируются в абзацы по детерминированному распределению длин (1–20 предложений). Абзацы разделяются пустой строкой.

Декодирование

Процесс обращается в обратном порядке. Текст нормализуется (лишние переводы строк схлопываются), затем с начала каждого фрагмента ищется самое длинное совпадение с предложением из корпуса. Из найденных предложений восстанавливается поток бит, снимается маркер конца, выполняется обратное перемешивание. Полученные байты при необходимости расшифровываются паролем GPG, либо выдаются как -----BEGIN PGP MESSAGE----- для вставки в Kleopatra / gpg (асимметричный сценарий).

Корпус предложений

Используется большой набор реальных русских предложений. Имена персонажей из исходных текстов заменены случайными русскими именами, чтобы скрытый текст выглядел естественнее. Корпус загружается в браузер один раз и дальше обрабатывается локально — ничего не отправляется на сервер.

Ограничения

  • Декодирование требует той же версии алгоритма, что использовалась при кодировании.
  • Декодирование требует точного совпадения предложений с корпусом: нельзя произвольно переписывать слова.
  • Чем больше данных, тем длиннее cover-текст — каждый байт кодируется примерно в одно предложение.
  • Стеганография не делает текст «незаметным» для анализа: при подозрении можно проверить статистику или попытаться декодировать.
  • Пароль GPG защищает содержимое, но не скрывает сам факт встраивания данных.

Все вычисления выполняются локально в браузере. Корпус предложений загружается один раз (~70 МБ).
